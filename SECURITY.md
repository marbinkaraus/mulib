# Security

## Reporting a vulnerability

If you believe you have found a **security vulnerability** in Mulib (for example in the app, its update process, or how it handles local data), please report it **privately** instead of opening a public issue.

**Preferred:** use [GitHub Security Advisories](https://github.com/marbinkaraus/mulib/security/advisories/new) for this repository.

Include:

- A short description of the issue and its impact  
- Steps to reproduce or a proof of concept, if you can share one safely  
- Affected version or commit, if known  

We will treat reports seriously and respond as capacity allows. This process is for **security defects in the software**, not for general bugs or legal questions about how the app is used.
